# Implementation Plan: Imprecise Multi-Armed Bandit Agent

**Date**: 2026-03-20 (updated)
**Based on**: Kosoy (2024), "Imprecise Multi-Armed Bandits" (arXiv:2405.05673)
**Reference impl**: [norabelrose/infrabayes](https://github.com/norabelrose/infrabayes)

---

## 1. Executive Summary

We implement a **faithful version of IUCB** (Imprecise Upper Confidence Bound) from
Kosoy's thesis for 2-action Newcomb-like games. Rather than simplifying away the
algorithm's structure, we preserve the cycle-based updates, the proper confidence
set geometry, and the norm definitions that the thesis's regret proofs depend on.
This gives us the formal O~(sqrt(N)) regret guarantee (Theorem 6.1) and
O~(ln N) in the positive-gap case (Theorem 6.3).

To support IUCB, we also refactor the environment/agent interface so that the
agent observes the full outcome tuple (environment's action, agent's action,
reward) rather than just the reward scalar. This eliminates the observation
model mismatch that would otherwise prevent us from applying the thesis's
framework.

## 2. Theory Overview

*(Sections 2.1--2.4 unchanged from previous version of this plan.)*

### 2.1 The Imprecise Bandit Model

- **Arms**: A compact subset A of R^{D_X}
- **Outcomes**: A compact convex set D in a vector space Y (not just a reward scalar)
- **Hypothesis**: A mapping H: A -> credal sets over D. When arm x is pulled, the
  outcome is drawn from some distribution in H(x); the exact distribution is
  adversarially chosen.
- **Lower prevision**: ME_H[r|x] = min_{zeta in H(x)} E_{y~zeta}[r(x,y)]
- **Regret**: ERg_H(phi; N) = N * ME_H[r|x*_H] - min_{nu in P_H} sum E[r(x_n, y_n)]

### 2.2 Linear Imprecise Bandits (Assumptions 1-4)

The thesis restricts to credal sets of the form:

    H(x) = { zeta in Delta(D) | E[y] in K(x)+ }

where K(x)+ = ker(F_{x,theta}) ∩ D, and F: A x Z x Y -> W is bilinear in (z,y).

**Key parameters**:
- D_Z = dim(Z), D_W = dim(W)
- R = max_{theta in H} ||theta||
- S = min_{x,theta} sin(K_theta(x)^flat, D)
- C = max(r) - min(r)

### 2.3 The IUCB Algorithm (Algorithm 4)

```
Input: eta in R+
C <- H
for k from 1 to infinity:
    theta* <- argmax_{theta in C} max_{x in A} ME_theta[r|x]
    tau <- 0, Sigma_y <- 0
    do:
        select arm x*_{theta*} and observe outcome y
        tau <- tau + 1
        Sigma_y <- Sigma_y + y
        y_bar <- Sigma_y / tau
    while sqrt(tau) * max_{theta in C} min_{z in V(x*_{theta*}, y_bar)} ||theta - z||
          < 2(D_Z + 1) * eta
    C <- { theta in C | min_{z in V(x*_{theta*}, y_bar)} ||theta - z|| <= eta/sqrt(tau) }
end for
```

### 2.4 Regret Bounds

**General case (Theorem 6.1)**:
    ERg = O~( R * S^{-1} * D_Z^2 * D_W^{5/6} * sqrt(N) + D_Z^2 * C )

**Positive gap case (Theorem 6.3)**:
    ERg = O~( D_Z^4 * R^2 * S^{-2} * D_W^{5/3} * ln(N) / g + D_Z^2 * C )

**For our 2x2 games** (Example 6.3): O~( (|B1|*|B2|)^{17/3} * ln(N) / g )

## 3. Mapping Thesis to Our Repo

### 3.1 Our Newcomb-like Environments as Imprecise Bandits

Our `BaseNewcombLikeEnvironment` with 2 actions maps to Example 4.3:

| Our concept | Thesis concept |
|---|---|
| `get_probabilities()` -> p | Agent's mixed strategy x in Delta(B1) |
| Environment samples prediction | Player 2's pure strategy b in B2 |
| Agent takes action a | Player 1's pure strategy a in B1 |
| `reward_table[i, j]` | Payoff matrix P* |
| Agent observes (prediction, action, reward) | Full outcome y in D |

For 2 actions: B1 = B2 = {0, 1}, D_Z effectively 4 (payoff matrix entries),
D_W ~ 6, R <= 12, S = 1, C <= 2.

### 3.2 Observation Model: The Fix

The thesis (Section 2.4) requires the agent to observe player 2's pure strategy
(the prediction) each round. Currently our environments hide this. We fix this
by refactoring the environment interface (see Section 5).

### 3.3 Classical Bandits as a Degenerate Case

`BanditEnvironment` is a stochastic bandit (Example 4.10) -- credal sets are
singletons, IUCB reduces to standard UCB.

## 4. Relationship to norabelrose/infrabayes

**Very little is directly reusable.** The norabelrose repo implements the
infra-Bayesian framework (sa-measures, infradistributions) from Kosoy's LessWrong
sequence. The thesis uses a different formalism: credal sets with linear
hypothesis classes, a frequentist confidence-set algorithm (not Bayesian updating),
and no temporal composition (bandits, not RL). Sa-measures are not needed because
there are no state transitions to compose over, and the algorithm maintains a
confidence set rather than updating a prior. See Section 4 in the previous
version of this plan for details.

## 5. Interface Refactor: `Outcome` and `step()`

### 5.1 Motivation

IUCB requires the agent to observe the full outcome tuple
(env_action, agent_action, reward) each round. The current interface
(`interact(action) -> float`) only returns the reward. Rather than hacking
a `(reward, prediction)` return into `interact()`, we refactor the interface
to cleanly separate the environment's internal causal structure (respond to
policy, then resolve payoff) from the external contract (one call, one
structured result).

### 5.2 The Outcome Type

```python
# ibrl/outcome.py

from dataclasses import dataclass
import numpy as np

@dataclass
class Outcome:
    """Result of one round of agent-environment interaction.

    Attributes:
        reward:     The scalar reward.
        env_action: The environment's move (e.g. the predictor's prediction).
                    None for environments with no environment move (standard bandits).
    """
    reward: float
    env_action: int | None = None
```

### 5.3 Refactored BaseEnvironment

```python
# ibrl/environments/base.py

class BaseEnvironment(ABC):

    def step(self, probabilities: NDArray[np.float64], action: int) -> Outcome:
        """Execute one round of interaction.

        The environment first responds to the agent's policy (e.g. the predictor
        samples a prediction), then resolves the payoff given the agent's action.

        Arguments:
            probabilities: The agent's policy (probability distribution over actions)
            action:        The action sampled from the policy

        Returns:
            Outcome containing the reward and any environment action
        """
        env_action = self._respond(probabilities)
        reward = self._resolve(env_action, action)
        return Outcome(reward=reward, env_action=env_action)

    def _respond(self, probabilities: NDArray[np.float64]) -> int | None:
        """Environment's move given the agent's policy. Override in subclasses.

        This is called before the payoff is determined. For Newcomb-like
        environments, this is where the predictor samples its prediction.
        For standard bandits, returns None.
        """
        return None

    def _resolve(self, env_action: int | None, action: int) -> float:
        """Determine the reward given both moves. Override in subclasses."""
        raise NotImplementedError

    # predict() and interact() removed from the base class
```

### 5.4 Refactored BaseAgent

```python
# ibrl/agents/base.py

class BaseAgent(ABC):

    def update(self, probabilities: NDArray[np.float64], action: int,
               outcome: Outcome) -> None:
        """Update internal state based on outcome of the round.

        Arguments:
            probabilities: The policy the agent declared
            action:        The action that was sampled
            outcome:       The full outcome (reward + env_action)
        """
        self.step += 1
```

### 5.7 Migration of Existing Agents

All existing agents use `reward` from `update()`. They just need to read
`outcome.reward` instead. Since they all call `super().update(...)`, the
signature change propagates from `BaseAgent`. The agents that don't use
`env_action` simply ignore it. Example for QLearningAgent:

```python
def update(self, probabilities, action, outcome):
    super().update(probabilities, action, outcome)
    reward = outcome.reward  # only change
    # ... rest unchanged
```

### 5.8 Backward Compatibility Note

This is a breaking change to the base class interfaces. All 7 agents and 7
environments need updating, plus the simulator and any notebooks. The changes
are mechanical (rename `reward` -> `outcome.reward`, restructure `predict` +
`interact` -> `_respond` + `_resolve`). There is no change to behavior -- all
existing agents will produce identical results.

## 6. Classic UCB Agent (Baseline)

### 6.1 Why We Need This

None of the existing agents implement UCB. The "classical" agent is Q-learning
with epsilon-greedy exploration; the Bayesian agent uses precision-weighted
updates with epsilon-greedy/softmax. These use random noise for exploration.

UCB (Upper Confidence Bound) explores structurally differently: it adds an
optimistic bonus to each arm's estimated value, proportional to uncertainty.
Arms that have been pulled less get a larger bonus, driving exploration without
any randomness in the selection rule. This is the *same principle* that IUCB
generalizes to the imprecise setting, so UCB is the natural baseline.

It also gives us a clean comparison:
- **UCB vs IUCB on standard bandits**: Should perform similarly (IUCB reduces
  to something UCB-like when credal sets are singletons).
- **UCB vs IUCB on Newcomb-like games**: UCB should fail (it treats arms
  independently, ignoring policy-dependence), while IUCB should excel.
- **UCB vs existing agents on standard bandits**: UCB should match or beat
  epsilon-greedy Q-learning, giving us confidence the implementation is correct.

### 6.2 Algorithm: UCB1

From the thesis (Algorithm 1) and Lattimore & Szepesvari (2020):

```
for every x in A:
    select arm x and receive reward r
    mu[x] <- r
    n[x] <- 1
end for
for every following round t:
    x* <- argmax_x  mu[x]/n[x] + c * sqrt(ln(t) / n[x])
    select arm x* and receive reward r
    mu[x*] <- mu[x*] + r
    n[x*] <- n[x*] + 1
end for
```

The exploration parameter c controls the width of the confidence bound.
The thesis uses c = 2 (for rewards in [-1, +1]). We make it configurable.

**Regret bound (Theorem 2.1)**: ERg <= 8 * sqrt(|A| * N * ln N) + 3|A|

### 6.3 Implementation

**File**: `ibrl/agents/ucb.py`

```python
class UCBAgent(BaseAgent):
    """Classic UCB1 agent.

    Selects arms by argmax of (empirical mean + confidence bound).
    No epsilon-greedy or softmax -- exploration is entirely driven
    by the confidence term.

    This agent does NOT inherit from BaseGreedyAgent because UCB's
    action selection is deterministic (argmax), not stochastic
    (epsilon-greedy/softmax). It returns a one-hot probability
    distribution from get_probabilities().

    Arguments:
        exploration: Confidence bound multiplier (default 2.0, matching
                     the thesis's analysis for rewards in [-1, +1]).
    """

    __init__(num_actions, seed, verbose, exploration=2.0)

    Attributes:
        q: ndarray(k,)      -- cumulative reward per arm
        counts: ndarray(k,) -- pull count per arm
        exploration: float   -- the constant c

    get_probabilities() -> ndarray:
        # Initialization: pull each arm once (round-robin)
        if any arm has count 0:
            return one-hot for next unpulled arm

        # UCB selection: deterministic argmax
        t = sum(counts)
        ucb_values = q / counts + exploration * sqrt(ln(t) / counts)
        best = argmax(ucb_values)  # break ties randomly
        return one-hot(best)

    update(probabilities, action, outcome):
        super().update(probabilities, action, outcome)
        self.counts[action] += 1
        self.q[action] += outcome.reward

    reset():
        super().reset()
        self.q = zeros(num_actions)
        self.counts = zeros(num_actions)

    dump_state():
        return string showing q/counts and ucb_values
```

**Design note**: UCB returns a deterministic (one-hot) policy from
`get_probabilities()`. This is correct -- UCB's action selection is
deterministic given the history. In Newcomb-like environments, the predictor
will see a near-deterministic policy and predict accordingly, which is part
of why UCB fails there (it can't play mixed strategies).

### 6.4 Registration

Add to `ibrl/agents/__init__.py` and to `construction.py`:
```python
"ucb": agents.UCBAgent
```

Invocable as `"ucb"` or `"ucb:exploration=1.0"`.

### 6.5 Expected Behavior Across Environments

| Environment | UCB expected behavior |
|---|---|
| `bandit` | Near-optimal. This is what UCB is designed for. |
| `newcomb` | Fails. Will two-box (converge to action 1) because it treats actions independently and sees higher per-action reward for two-boxing. |
| `damascus` | Fails. Will oscillate or lock onto one action rather than finding the p=0.5 equilibrium, since UCB always plays deterministically. |
| `asymmetric-damascus` | Fails similarly to Damascus. |
| `coordination` | May find the better equilibrium by luck (if it tries action 0 first and gets higher reward), but not reliably. |
| `switching` | Fails after the switch. UCB's confidence bounds only shrink; it can't track non-stationarity. |

These failure modes on Newcomb-like environments are exactly what IUCB is
designed to fix, making UCB the ideal baseline for demonstrating IUCB's value.

## 7. Full IUCB Implementation

### 7.1 Why Not Simplify

The previous version of this plan proposed simplifications (interval confidence
sets, per-round updates, no cycles) that break the theoretical guarantees. The
thesis's proofs depend on:

1. **Cycle-based updates**: Concentration bounds (Lemma A.3) require tau_k IID
   samples from the same arm. Per-round updates with changing arms break the
   martingale structure.
2. **The stopping condition**: Ensures each cycle reduces confidence set volume
   by a constant factor, bounding the number of "wasteful" cycles to O(D_Z^2)
   via John's ellipsoid theorem.
3. **Confidence set geometry**: Slab-based updates (distance to affine subspaces
   in Z_bar) enable the volumetric argument. Interval-based updates don't.

For 2x2 games, the full algorithm is tractable: all optimization happens in
~4-10 dimensions, well within reach of scipy/cvxpy.

### 7.2 Concrete Objects for the 2x2 Game Case (Example 4.3)

We need to work out Z, W, F, and the norms for |B1| = |B2| = 2.

**Outcome space**: B = B2 x B1 x {-1,+1}. With B1=B2={0,1}, |B| = 8.
Y = R^B = R^8, D = Delta(B) (the 7-simplex).

**Hypothesis space**: From Example 4.3 and 4.9:
- For each b in B2: Z_b = R, psi_b = 1, theta_b = 1 (fixed, not free)
- For each (b,a) in B2 x B1: Z_{ba} = R^2, psi_{ba} = [1, 1],
  theta_{ba} = [(1-P[b,a])/2, (1+P[b,a])/2]
- Free parameters: P[0,0], P[0,1], P[1,0], P[1,1] in [-1, 1]

So Z = R^2 + R^8 = R^10, but effectively 4 free parameters (the payoff matrix
entries). The full D_Z = dim(Z) = 10 in the thesis's notation, but after
factoring out the fixed structural parameters (theta_b = 1 for all b in B2),
the confidence set lives in a 4D subspace. We work in this projected 4D space
throughout the implementation. When the regret bounds reference D_Z, use 10
(the formal dimension); when implementing the confidence set operations, use
the 4D effective parameterization. H = [-1,1]^4 (the box of valid payoff
matrices).

**The map F**: F(x, z, y) as defined in Example 4.9. For the 2x2 case this
can be written out as a concrete matrix operation. This is the key derivation
work.

**Norms**: The norm on Y is l1 (Lemma E.1). The norm on W and the operator
norm on Z_bar are derived from F and the arm set A = [0,1] (the 1-simplex of
mixed strategies).

### 7.3 Phase 1: Derive and Implement the Concrete Algebra

**File**: `ibrl/iucb/algebra.py`

Implement the concrete instances of the thesis's abstract objects for Example 4.3
with |B1|=|B2|=2:

```
Functions:
    F(x, z, y) -> w
        The bilinear map for our 2x2 game case.
        x: float in [0,1] (agent's mixing probability)
        z: ndarray(10,) (hypothesis vector in Z)
        y: ndarray(8,) (outcome vector in Y)
        Returns: ndarray in W

    F_bar(x, z_bar, y) -> w
        The extended map on Z_bar.

    theta_from_payoff(P) -> ndarray(10,)
        Convert a 2x2 payoff matrix to the thesis's theta representation.

    payoff_from_theta(theta) -> ndarray(2,2)
        Inverse of above.

    z_bar_norm(z_bar) -> float
        ||z_bar|| = max_{x in [0,1]} ||F_bar_{x,z_bar}||_op
        1D maximization of operator norm.

    w_norm(w) -> float
        ||w|| = max_{x,theta} min_{y: F_{x,theta}(y)=w} ||y||

    dist_to_subspace(theta, V) -> float
        min_{z in V} ||theta - z|| in the Z_bar norm.
        V is an affine subspace (ker F_bar_{x,y}).
        Convex optimization problem.

    V(x, y_bar) -> AffineSubspace
        ker F_bar_{x, y_bar} -- the affine subspace used in confidence
        set updates.

    lower_prevision(theta, x, r) -> float
        ME_theta[r|x] = min_{y in K_theta(x)+} r(x, y)
        For the 2x2 game: the game value for row player strategy x
        against payoff matrix P(theta).
```

### 7.4 Phase 2: Confidence Set

**File**: `ibrl/iucb/confidence_set.py`

```
ConfidenceSet:
    """Maintains the confidence set C_k as the intersection of:
    - The initial hypothesis set H = [-1,1]^4 (in payoff space)
    - A list of slab constraints from each completed cycle

    Each slab constraint is:
        { theta | dist_{Z_bar}(theta, V_k) <= r_k }
    where V_k and r_k come from cycle k's observations.
    """

    __init__(H: initial hypothesis set as box bounds)

    contains(theta) -> bool:
        Check all constraints.

    max_dist_to_subspace(V) -> float:
        max_{theta in C} dist(theta, V)
        Maximizing a convex function over a convex set.
        For 4D, use scipy.optimize or sample extreme points.

    update(V, radius):
        Add a new slab constraint.

    optimistic_theta(arm_set, reward_fn) -> theta:
        argmax_{theta in C} max_{x in A} ME_theta[r|x]
        For 2x2: maximize game_value(P(theta)) over theta in C.
```

The confidence set lives in the 4D payoff-matrix space (not the full 10D Z).
We project everything to this effective subspace.

### 7.5 Phase 3: IUCB Agent

**File**: `ibrl/agents/iucb.py`

```
IUCBAgent(BaseAgent):
    """Faithful implementation of IUCB (Algorithm 4) for 2-action games.

    Implements the full cycle-based algorithm with proper confidence set
    geometry, preserving the theoretical guarantees of Theorems 6.1 and 6.3.
    """

    __init__(num_actions, seed, verbose, eta=None, time_horizon=None)
        # eta can be set automatically from time_horizon using Theorem 6.1's
        # prescription: eta = Theta(1) * R * D_W^{5/6} * sqrt(ln(C*D_W*N))
        # If time_horizon is not known, use doubling trick.

    Attributes:
        confidence_set: ConfidenceSet
        current_theta: ndarray  -- the optimistic hypothesis for current cycle
        current_policy: ndarray -- the maximin policy for current_theta
        cycle_outcomes: list    -- outcome vectors y collected in current cycle
        tau: int                -- rounds in current cycle so far
        cycle_count: int

    get_probabilities() -> ndarray:
        Return self.current_policy (fixed within a cycle).

    update(probabilities, action, outcome):
        1. Construct the outcome vector y from (outcome.env_action, action,
           outcome.reward). This is the one-hot-ish vector in D = Delta(B).
        2. Append y to cycle_outcomes, increment tau.
        3. Compute y_bar = mean(cycle_outcomes).
        4. Compute V = V(x*_{current_theta}, y_bar).
        5. Check stopping condition:
           sqrt(tau) * confidence_set.max_dist_to_subspace(V) >= 2(D_Z+1)*eta
        6. If met:
           a. Update confidence set: add slab { dist(theta, V) <= eta/sqrt(tau) }
           b. Select new optimistic theta from updated confidence set
           c. Compute new maximin policy for new theta
           d. Reset cycle state (tau=0, cycle_outcomes=[])
        7. If not met: continue pulling same arm next round.

    _start_new_cycle():
        self.current_theta = self.confidence_set.optimistic_theta(...)
        self.current_policy = solve_game(payoff_from_theta(self.current_theta))
        self.cycle_outcomes = []
        self.tau = 0

    _outcome_to_y(env_action, action, reward) -> ndarray(8,):
        Construct the outcome vector y in D = Delta(B2 x B1 x {-1,+1}).
        For a single observation, this is a point in D (not a vertex of the
        simplex, because the reward is real-valued, not binary).

        Following the thesis's encoding (end of Example 4.3):
        y_{b,a,+1} = (1+r)/2  if (b,a) matches (env_action, action)
        y_{b,a,-1} = (1-r)/2  if (b,a) matches (env_action, action)
        y_{b,a,sigma} = 0     otherwise

        This encodes the observation that env_action=b, action=a, and the
        reward r has the given value.
```

## 9. Limitations and Concerns

### 9.1 Computational Complexity

The thesis notes (Section 9) that computational complexity of IUCB is an open
question in general. For our 2x2 case:
- Confidence set operations are convex programs in 4D -- trivial.
- Optimistic theta selection maximizes game value over a 4D convex set -- modest.
- The stopping condition requires one convex max per round within a cycle.

All feasible with scipy/cvxpy. No concern for 2x2.

### 9.2 Non-Anytime Algorithm

IUCB requires knowing N to set eta optimally. We can use a **doubling trick**:
start with N_0 = some small value, run IUCB with that horizon, then double N_0
and restart. This costs a constant multiplicative factor in regret. The thesis
lists "make it anytime" as future work.

### 9.3 Extension to k > 2 Actions

The algebra of Section 7.2 generalizes to k > 2 with |B| = k^2 * 2, D_Z
growing as k^2. The optimistic selection becomes harder (game value is no
longer closed-form; need LP). The confidence set operations remain convex
but in higher dimensions. Feasible for moderate k but not the focus of this
plan.

### 9.4 The Parameter S

S = 1 for our setting (Proposition 8.9), the best possible. No geometric
blowup in regret bounds. This is specific to the partial conditional bandit
structure (Example 4.2) and would not hold for arbitrary imprecise bandits.

## 10. Theoretical Assumptions

1. **Stationary payoff matrix**: Holds for all Newcomb-like environments.
   Does not hold for `switching` -- IUCB is not designed for that.

2. **Full outcome observation**: After the interface refactor, the agent
   observes (env_action, action, reward), matching Example 4.3 exactly.

3. **Known reward range**: Rewards lie in a known bounded interval. True for
   all our environments.

4. **Predictor samples from declared policy**: This is what
   `BaseNewcombLikeEnvironment._respond()` does. Matches the thesis's model
   where the adversary sees the agent's mixed strategy.

## 11. Remaining Unclear Aspects

### 11.1 Optimistic Optimization Implementation

argmax_{theta in C} max_x ME_theta[r|x] is a non-concave maximization
(game value is neither convex nor concave in P). For 4D with a modest number
of constraints, we can use:
- Multi-start local optimization (scipy.optimize.minimize with negated objective)
- Grid search over the 4D confidence set followed by local refinement
- Branch and bound

This is an engineering question, not a theoretical obstacle.

### 11.2 The Exact Form of V(x, y_bar)

V(x, y) = ker F_bar_{x,y} is an affine subspace of Z_bar. For the 2x2 case
this is a concrete linear algebra computation. The derivation is the main work
of Phase 1.

### 11.3 Handling the First Cycle

The algorithm starts with C_0 = H and must find an optimistic theta. On the
very first cycle, the confidence set is the full box [-1,1]^4. The optimistic
theta will be the one with the highest game value in this box.

## 12. Will an Imprecise Bandit Agent Shine vs. Classic MAB?

**For Newcomb-like 2-action games: yes.** Classic MAB agents (Q-learning,
Bayesian, EXP3) treat each action independently and cannot represent the
correlation between policy and reward. IUCB reasons about the full payoff
matrix and optimizes over mixed strategies, which is exactly what these
environments require.

- **Newcomb**: IUCB will one-box (game value is 10 for one-boxing)
- **Damascus**: IUCB converges to p=0.5 (minimax, value=5)
- **Coordination**: IUCB finds the better equilibrium

**For standard bandits: comparable performance.** Credal sets collapse to
singletons, IUCB reduces to something UCB-like.

**For adversarial/switching environments: not the right tool.** IUCB assumes
stationarity. EXP3 is better here.

**Does infra-Bayesian reasoning need full RL to shine?** For the *current*
environments (stateless, repeated), bandit-level reasoning suffices because
the hypothesis space is simple. The real power of infra-Bayesian methods
would emerge in environments with state transitions, model uncertainty, and
adversarial dynamics -- that requires RL, not bandits. The bandit setting
is a proving ground.

### 15.2 Issues and Deviations from Plan

**1. The eta parameter is impractical at theoretical scale.**

The plan (Section 9.1) noted computational complexity as a potential concern
but concluded "No concern for 2x2." This turned out to be wrong -- not
because of optimization dimensionality, but because of the *stopping
condition scale*.

The thesis's eta prescription (Theorem 6.1) is:

    eta = Theta(1) * R * D_W^{5/6} * sqrt(ln(C * D_W * N))

For 2x2 games: R=12, D_W=6, giving eta ~ 200. The stopping condition
threshold is 2*(D_Z+1)*eta ~ 4400. Since max_dist from any hypothesis to
the kernel V is O(1), each cycle requires tau ~ 4400^2 ~ 20M rounds.

The exponential penalty term in Theorem 6.1 requires eta^2 >> R^{5/2} *
D_W^3 ~ 10^6 to be negligible, confirming that the theoretically correct
eta is enormous. The formal regret guarantee only kicks in at astronomically
large N.

**Resolution**: We use a heuristic eta = N^{1/4} / (2*(D_Z+1)), which
gives ~sqrt(N) cycles over N rounds. This sacrifices the formal guarantee
at finite N but makes the algorithm functional. The plan should have flagged
this as a fundamental limitation of the 2x2 case constants, not just an
engineering question.

**2. IUCB two-boxes on Newcomb (Section 12 claim is wrong).**

The plan states: "Newcomb: IUCB will one-box (game value is 10 for
one-boxing)." This is incorrect. The minimax game value for Newcomb is:

    max_x min_b sum_a x[a] * P[b,a]

For the Newcomb payoff matrix [[10,15],[0,5]]:
- x=[1,0] (one-box): min(10, 0) = 0
- x=[0,1] (two-box): min(15, 5) = 5
- Game value = 5, achieved by two-boxing

IUCB models the predictor as adversarial (the credal set K_theta(x) allows
any distribution over B2, not just copying x). This is the CDT answer.
The framework cannot exploit the cooperative structure of the predictor
(that it samples from the agent's declared strategy).

Corrected expected behavior:

| Environment | IUCB expected behavior |
|---|---|
| `newcomb` | Two-boxes (minimax value 5, not 10). CDT answer. |
| `damascus` | Converges to p=0.5 mixed strategy (minimax value 5). Correct. |
| `coordination` | Finds mixed equilibrium (minimax), not the pure equilibrium. |
| `bandit` | Near-optimal, reduces to UCB-like. |

This is not a bug -- it's a property of the imprecise bandit framework.
The framework is conservative by design. Exploiting the Newcomb correlation
would require the agent to model the predictor's strategy as a function of
its own, which is outside the scope of the adversarial credal set model.

**3. UCB does not fail on Newcomb the way the plan predicted.**

Section 6.5 states UCB "Will two-box (converge to action 1) because it
treats actions independently and sees higher per-action reward for
two-boxing." In fact, UCB one-boxes on Newcomb (getting reward 10)! Because
UCB plays deterministic policies, the predictor always matches its action,
so UCB only ever sees diagonal entries of the reward table:
reward_table[0,0]=10 vs reward_table[1,1]=5. It correctly picks action 0.

UCB *does* fail on Damascus (reward 0, optimal 5) because deterministic
policies always result in "death" on the anti-diagonal payoff structure.

**4. Performance optimization was necessary.**

The plan didn't discuss computational cost per step. The initial
implementation used `scipy.optimize.differential_evolution` (a global
optimizer) inside `max_dist_to_V`, called on every round within a cycle.
This was far too slow (~60s for 100 steps).

Optimizations applied:
- Replaced `differential_evolution` with corner evaluation (2^4=16 corners
  of [-1,1]^4) + L-BFGS-B local refinement
- Added early-skip: don't check stopping condition when sqrt(tau) is too
  small to possibly trigger it
- Added periodic checking: only evaluate the stopping condition every
  max(1, tau//10) rounds within a cycle

Final performance: ~6s for 500 steps on Damascus (acceptable).

**5. The switching environment needed a variable rename.**

The `SwitchingAdversaryEnvironment` used `self.step` as an internal counter,
which conflicted with the new `step()` method name. Renamed to
`self._step_count`.
